**1. Create `crates/smelt-core/src/summary/scope.rs`** with:

- `check_scope(file_scope, shared_files, changed_files) -> Vec<ScopeViolation>`:
  ```rust
  /// Check which files are out of scope for a session.
  ///
  /// Returns an empty Vec if `file_scope` is `None` — scope checking is opt-in.
  /// A file is in-scope if it matches any `file_scope` glob OR any `shared_files` glob.
  /// All other files are violations.
  pub fn check_scope(
      session_name: &str,
      file_scope: Option<&[String]>,
      shared_files: &[String],
      changed_files: &[String],
  ) -> Vec<ScopeViolation>
  ```
  Implementation:
  1. If `file_scope` is `None`, return empty Vec immediately (short-circuit — everything is in scope).
  2. Build a `GlobSet` from all `file_scope` patterns + all `shared_files` patterns. Use `globset::GlobSetBuilder`.
  3. For each changed file, check if it matches the GlobSet. If not, create a `ScopeViolation`.
  4. The `ScopeViolation.file_scope` field should contain the session's `file_scope` patterns (not shared_files — those are always permitted).

- `build_scope_matcher(file_scope, shared_files) -> Result<GlobSet>`:
  ```rust
  /// Build a GlobSet from file_scope and shared_files patterns.
  ///
  /// Callers should have already validated globs via Manifest::validate().
  /// This function still returns a Result for robustness.
  fn build_scope_matcher(
      file_scope: &[String],
      shared_files: &[String],
  ) -> crate::Result<globset::GlobSet>
  ```
  Implementation:
  1. Create `GlobSetBuilder`
  2. Add each pattern from `file_scope` and `shared_files` via `globset::Glob::new(pattern)?.build()?`
  3. Call `builder.build()` and map errors to `SmeltError::Orchestration`

- Unit tests:
  - `no_file_scope_means_no_violations` — file_scope=None, several changed files, expect empty violations
  - `all_files_in_scope` — file_scope=Some(["src/**"]), changed_files=["src/main.rs", "src/lib.rs"], expect empty violations
  - `out_of_scope_files_detected` — file_scope=Some(["src/auth/**"]), changed_files=["src/auth/login.rs", "src/db/schema.rs"], expect 1 violation for db/schema.rs
  - `shared_files_override_scope` — file_scope=Some(["src/auth/**"]), shared_files=["Cargo.toml", "Cargo.lock"], changed_files=["src/auth/login.rs", "Cargo.toml"], expect 0 violations
  - `multiple_file_scope_patterns` — file_scope=Some(["src/auth/**", "src/lib.rs"]), changed_files=["src/auth/login.rs", "src/lib.rs", "src/db/mod.rs"], expect 1 violation for db/mod.rs
  - `empty_file_scope_means_all_violations` — file_scope=Some([]) (empty vec, not None), changed_files=["a.txt"], expect 1 violation (edge case: explicit empty scope)
  - `violation_captures_session_file_scope` — verify the ScopeViolation.file_scope field contains the original file_scope patterns
