  <name>Task 1: Implement remove, orphan detection, and prune in WorktreeManager</name>
  <files>
    crates/smelt-core/src/worktree/mod.rs
    crates/smelt-core/src/worktree/orphan.rs
  </files>
  <action>
    1. Create `worktree/orphan.rs` with orphan detection logic:
       - `pub fn is_pid_alive(pid: u32) -> bool` — use `unsafe { libc::kill(pid as i32, 0) == 0 }` (standard POSIX pattern)
       - `pub fn is_likely_orphan(state: &WorktreeState, git_worktrees: &[GitWorktreeEntry], staleness_threshold: chrono::Duration) -> bool`
         - Checks: (a) PID is set AND not alive, OR (b) status is Running AND updated_at is older than staleness_threshold, OR (c) state has a worktree_path but no matching entry in git_worktrees (state desync)
         - Does NOT mark Created/Completed/Failed as orphaned — only Running sessions can become orphaned
       - Use a default staleness threshold of 24 hours (per RESEARCH.md recommendation)
       - Unit tests: test is_pid_alive with PID 1 (init, always alive), test with PID 0 or very large PID (should be false), test is_likely_orphan with various state combinations

    2. Add `pub mod orphan;` to `worktree/mod.rs`.

    3. Implement `WorktreeManager::remove(&self, name: &str, force: bool, skip_confirm: bool) -> Result<RemoveResult>`:
       - `RemoveResult`: enum or struct indicating what was removed (worktree, branch, state file)
       - Steps:
         a. Load `.smelt/worktrees/{name}.toml` — return `WorktreeNotFound` if missing
         b. Resolve worktree_path to absolute (it's stored relative to repo_root)
         c. Check if worktree is dirty via `git.worktree_is_dirty(&abs_path)` — if dirty and not force, return `WorktreeDirty`
         d. Remove worktree: `git.worktree_remove(&abs_path, force)`
         e. Check branch merge status: `git.branch_is_merged(&state.branch_name, "HEAD")` — if not merged and not force, return `BranchUnmerged`
         f. Delete branch: `git.branch_delete(&state.branch_name, force)` — use force=true if force was passed, otherwise force=false (which uses -d, failing on unmerged)
         g. Remove state file: `std::fs::remove_file`
         h. Run `git.worktree_prune()` to clean up stale metadata

    4. Implement `WorktreeManager::detect_orphans(&self) -> Result<Vec<(String, WorktreeState)>>`:
       - Read all state files from `.smelt/worktrees/`
       - Get git worktree list
       - For each state file, call `is_likely_orphan`
       - Return list of (session_name, state) pairs that are orphaned

    5. Implement `WorktreeManager::prune(&self, force: bool, skip_confirm: bool) -> Result<Vec<String>>`:
       - Call detect_orphans
       - For each orphan, call remove with force=true (orphans are already dead)
       - Return list of pruned session names
       - If no orphans found, return empty vec

    6. Unit tests:
       - Test remove cleans up worktree + branch + state file
       - Test remove with dirty worktree and force=false returns WorktreeDirty
       - Test remove with force=true succeeds on dirty worktree
       - Test detect_orphans identifies orphaned sessions (create a state file with a dead PID)
  </action>

  </verify>
  <done>
    WorktreeManager.remove() fully cleans up worktree + branch + state. Orphan detection works with PID + staleness + git cross-reference. Prune removes all detected orphans.
  </done>
