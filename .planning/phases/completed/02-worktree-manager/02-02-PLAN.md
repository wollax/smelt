    1. In `worktree/mod.rs`, create `WorktreeManager<G: GitOps>`:
       ```
       pub struct WorktreeManager<G: GitOps> {
           git: G,
           repo_root: PathBuf,
           smelt_dir: PathBuf,
       }
       ```
       Constructor: `new(git: G, repo_root: PathBuf)` â€” derives `smelt_dir` as `repo_root.join(".smelt")`.

    2. Implement `create(&self, opts: CreateWorktreeOpts) -> Result<WorktreeInfo>`:
       - `CreateWorktreeOpts`: session_name (String), base (String, default "HEAD"), dir_name (Option<String>), task_description (Option<String>), file_scope (Option<Vec<String>>)
       - `WorktreeInfo`: session_name, branch_name, worktree_path, base_ref, status
       - Validation steps:
         a. Check `.smelt/` exists, return `NotInitialized` if not
         b. Check `.smelt/worktrees/<name>.toml` doesn't exist, return `WorktreeExists` if so
         c. Check branch doesn't exist via `git.branch_exists("smelt/<name>")`, return `BranchExists` if so
         d. Compute worktree path: get repo directory name from repo_root, build `{repo_name}-smelt-{session_name}` (or use dir_name override), place as sibling to repo root (repo_root.parent().join(dir_name))
         e. Check worktree path doesn't already exist on disk
       - Create: call `git.worktree_add(&worktree_path, &branch_name, &base)` with branch_name = `smelt/{session_name}`
       - Write state: create `.smelt/worktrees/` dir if missing, write `WorktreeState` to `.smelt/worktrees/{session_name}.toml`
       - Store worktree_path as relative to repo_root in the TOML (use `pathdiff::diff_paths` or manual relative path computation â€” since it's always `../{dirname}`, compute it directly as `PathBuf::from("..").join(&dir_name)`)
       - Return `WorktreeInfo`

    3. Implement `list(&self) -> Result<Vec<WorktreeInfo>>`:
       - Read all `.smelt/worktrees/*.toml` files
       - For each, deserialize `WorktreeState`
       - Cross-reference with `git.worktree_list()` for consistency
       - Return sorted by session_name

    4. Add helper for state file I/O in `state.rs`:
       - `WorktreeState::load(path: &Path) -> Result<Self>` â€” read file, parse TOML, map toml errors to `SmeltError::StateDeserialization`
       - `WorktreeState::save(&self, path: &Path) -> Result<()>` â€” serialize to TOML, write file

    5. Update `init.rs`: create `.smelt/worktrees/` directory during `init_project()` (add after config.toml write).

    6. Update `lib.rs`: re-export `WorktreeManager` and key types from `worktree` module.

    7. Unit tests for WorktreeManager:
       - Test create writes state file and calls git correctly (can use a real git repo in tempdir, similar to existing cli.rs tests)
       - Test create with duplicate name returns WorktreeExists
       - Test list returns created worktrees
       - IMPORTANT: worktree paths are siblings to the temp repo â€” ensure test cleanup removes them. Use a wrapper that cleans up sibling worktree dirs on drop, or manually remove in the test.
  </action>
