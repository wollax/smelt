       - `GitWorktreeEntry` struct: path (PathBuf), head (String), branch (Option<String>), is_bare (bool), is_locked (bool) — derive Debug/Clone
       - `parse_porcelain(output: &str) -> Vec<GitWorktreeEntry>` function following the implementation in RESEARCH.md
       - Unit tests: test SessionStatus round-trip serde, test WorktreeState round-trip to TOML string, test parse_porcelain with sample porcelain output (normal, bare, detached HEAD)

<task type="auto">
  <name>Task 2: Extend GitOps trait and implement in GitCli</name>
  <files>
    crates/smelt-core/src/git/mod.rs
    crates/smelt-core/src/git/cli.rs
  </files>
  <action>
    1. Extend the `GitOps` trait in `git/mod.rs` with these methods (use `impl Future<Output = Result<...>> + Send` return style, consistent with existing methods):
       - `worktree_add(&self, path: &Path, branch_name: &str, start_point: &str) -> Result<()>` — runs `git worktree add -b <branch> <path> <start_point>`
       - `worktree_remove(&self, path: &Path, force: bool) -> Result<()>` — runs `git worktree remove <path>`, with `--force` if force=true
       - `worktree_list(&self) -> Result<Vec<GitWorktreeEntry>>` — runs `git worktree list --porcelain`, parses with `parse_porcelain`
       - `worktree_prune(&self) -> Result<()>` — runs `git worktree prune`
       - `worktree_is_dirty(&self, path: &Path) -> Result<bool>` — runs `git -C <path> status --porcelain`, returns true if non-empty output
       - `branch_delete(&self, branch_name: &str, force: bool) -> Result<()>` — runs `git branch -d` or `-D` based on force flag
       - `branch_is_merged(&self, branch_name: &str, base_ref: &str) -> Result<bool>` — checks if branch appears in `git branch --merged <base_ref>` output
       - `branch_exists(&self, branch_name: &str) -> Result<bool>` — runs `git rev-parse --verify refs/heads/<branch_name>`, returns true if exit code 0

    Import `GitWorktreeEntry` from `crate::worktree::state` in the git module.

    2. Implement all methods in `GitCli` (`git/cli.rs`):
       - For `worktree_is_dirty`: use `Command::new(&self.git_binary).args(["-C", path_str, "status", "--porcelain"])` — do NOT use `self.run()` since it uses `current_dir(&self.repo_root)`. Create a helper or inline the command.
       - For `branch_exists`: don't use `self.run()` either — the failure case (branch doesn't exist) returns exit code != 0, which `self.run()` would convert to an error. Run the command directly and check exit status.
       - For `worktree_list`: call `self.run(&["worktree", "list", "--porcelain"])` then parse with `parse_porcelain`.

    3. Add unit tests in `cli.rs` (extend `setup_test_repo` pattern):
       - Test `worktree_add` creates a worktree and `worktree_list` returns it
       - Test `worktree_remove` removes the worktree
       - Test `branch_exists` returns true for existing branch, false for non-existent
       - Test `branch_delete` removes a branch
       - Test `worktree_is_dirty` returns false for clean worktree, true after creating an untracked file
       - Test `branch_is_merged` for merged and unmerged branches

    NOTE: worktree tests create sibling directories outside the tempdir. Use `tempfile::Builder::new().prefix("smelt-wt-").tempdir()` for the worktree path, or construct a sibling path from the temp dir and ensure cleanup in the test.
  </action>
