**1. Create `crates/smelt-core/src/orchestrate/executor.rs`** with the Orchestrator:

```rust
pub struct Orchestrator<G: GitOps + Clone> {
    git: G,
    repo_root: PathBuf,
}
```

- `Orchestrator::new(git: G, repo_root: PathBuf) -> Self`

- `Orchestrator::run()` — the full lifecycle method:
  ```rust
  pub async fn run<H: ConflictHandler>(
      &self,
      manifest: &Manifest,
      manifest_content: &str,  // raw TOML for hash computation
      opts: &OrchestrationOpts,
      conflict_handler: &H,
      cancel: CancellationToken,
      on_status: impl Fn(&str, &SessionRunState) + Send + Sync,  // callback for dashboard updates
  ) -> crate::Result<OrchestrationReport>
  ```

  Implementation steps:
  1. **Validate & build DAG:** Call `build_dag(manifest)`. Determine `FailurePolicy` from `manifest.manifest.on_failure`.
  2. **Initialize run state:** Generate run_id via `RunState::generate_run_id`. Create RunState with all sessions as Pending. Save initial state.
  3. **Phase 1 — Create worktrees (sequential):** For each session in the manifest, create a worktree via `WorktreeManager::create()`. This is sequential to avoid git index lock contention. Update session state to indicate worktree created. If any worktree creation fails, mark that session as Failed, apply failure policy.
  4. **Phase 2 — Execute sessions (parallel via JoinSet):**
     - Use `tokio::task::JoinSet` for parallel session dispatch.
     - Main loop:
       a. Compute `ready_set()` from the DAG
       b. For each ready session, spawn a task into the JoinSet:
          - Create child CancellationToken
          - Run session: create `ScriptExecutor`, call `execute()` within `tokio::select!` against child token cancellation
          - Capture stdout/stderr to log file (via the ScriptExecutor's existing output)
          - Return (session_name, SessionResult)
       c. `tokio::select!` between `join_set.join_next()` and `cancel.cancelled()`:
          - On completion: update RunState (Completed/Failed), save state, apply failure policy
          - On cancellation: abort all tasks, mark remaining as Cancelled, save state, return early
       d. If a session fails:
          - SkipDependents: call `mark_skipped_dependents()`, mark those sessions as Skipped in RunState, call `on_status` for each, save state
          - Abort: cancel all via parent CancellationToken, return error
       e. Loop until JoinSet is empty and ready_set is empty
     - Handle `JoinError`: match `is_cancelled()` vs `is_panic()`. Map panics to Failed with panic message. **Never unwrap JoinSet results.**
  5. **Phase 3 — Merge (sequential via existing MergeRunner):**
     - Update RunState.phase to Merging, save
     - Filter manifest to only Completed sessions (create a filtered manifest clone)
     - If no completed sessions, skip merge phase
     - Create `MergeRunner`, call `run()` with the filtered manifest, merge opts, and conflict handler
     - Update RunState.phase to Complete, save
  6. **Build and return OrchestrationReport**

- `Orchestrator::resume()` — resume from incomplete run:
  ```rust
  pub async fn resume<H: ConflictHandler>(
      &self,
      manifest: &Manifest,
      manifest_content: &str,
      run_state: RunState,  // previously loaded incomplete state
      opts: &OrchestrationOpts,
      conflict_handler: &H,
      cancel: CancellationToken,
      on_status: impl Fn(&str, &SessionRunState) + Send + Sync,
  ) -> crate::Result<OrchestrationReport>
  ```
  - If RunState.phase is Sessions:
    - Skip sessions already in terminal state (Completed/Failed/Skipped/Cancelled)
    - Re-run from Phase 2 for remaining Pending sessions
  - If RunState.phase is Merging:
    - Skip directly to Phase 3 (merge)
    - Use merge_progress to determine what was already merged

- **Filtering manifest for merge:** Create a helper that builds a new `Manifest` containing only sessions where RunState shows `Completed`. This manifest is passed to `MergeRunner::run()`.

- **Session execution detail:** For each session task spawned into JoinSet:
  - The session needs its own ScriptExecutor instance. Since ScriptExecutor takes a `&G` reference and a worktree path, and the git ops are Clone, clone `self.git` for each spawned task.
  - Look up the session's worktree path from WorktreeState (loaded from `.smelt/worktrees/<session>.toml`)
  - If the session has no script, immediately return Completed (matching SessionRunner behavior)
  - Measure duration via `tokio::time::Instant`

- **Important:** Do NOT re-implement session execution logic. Reuse `ScriptExecutor` directly. The Orchestrator's value is in the DAG scheduling, not in session execution.
