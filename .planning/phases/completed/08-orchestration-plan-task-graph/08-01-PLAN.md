**2. Create `crates/smelt-core/src/orchestrate/types.rs`** with:

- `FailurePolicy` enum (Serialize, Deserialize, Default, Clone, Copy):
  - `SkipDependents` (default) — dependents of failed sessions are skipped
  - `Abort` — any failure stops the entire orchestration
  - `serde(rename_all = "kebab-case")`
  - `impl From<Option<&str>>` to convert from manifest's `on_failure` string

- `SessionRunState` enum (Serialize, Deserialize, Clone, Debug):
  - `Pending`
  - `Running`
  - `Completed { duration_secs: f64 }`
  - `Failed { reason: String }`
  - `Skipped { reason: String }`
  - `Cancelled`
  - `serde(tag = "status", rename_all = "lowercase")`
  - Helper methods: `is_terminal() -> bool` (Completed|Failed|Skipped|Cancelled), `is_success() -> bool` (Completed only)

- `RunPhase` enum (Serialize, Deserialize, Clone, Copy, Debug, PartialEq, Eq):
  - `Sessions` — executing sessions
  - `Merging` — running merge pipeline
  - `Complete` — all done successfully
  - `Failed` — orchestration failed
  - `serde(rename_all = "lowercase")`

- `MergeProgress` struct (Serialize, Deserialize, Clone, Debug):
  - `sessions_merged: Vec<String>` — sessions successfully merged so far
  - `current_session: Option<String>` — session currently being merged

- `RunState` struct (Serialize, Deserialize, Clone, Debug):
  - `run_id: String`
  - `manifest_name: String`
  - `manifest_hash: String` — SHA256 hex of manifest content (for resume validation)
  - `started_at: chrono::DateTime<chrono::Utc>`
  - `updated_at: chrono::DateTime<chrono::Utc>`
  - `phase: RunPhase`
  - `sessions: std::collections::HashMap<String, SessionRunState>`
  - `merge_progress: Option<MergeProgress>`
  - `failure_policy: FailurePolicy`
  - Methods:
    - `new(run_id, manifest_name, manifest_hash, failure_policy, session_names) -> Self` — initializes all sessions as Pending
    - `save(dir: &Path) -> crate::Result<()>` — writes `state.json` to the given run directory
    - `load(dir: &Path) -> crate::Result<Self>` — reads `state.json` from run directory
    - `is_complete() -> bool` — phase is Complete
    - `is_resumable() -> bool` — phase is Sessions or Merging (not Complete or Failed)
    - `generate_run_id(manifest_name: &str) -> String` — `<manifest_name>-<YYYYMMDD-HHMMSS>` format

- `OrchestrationOpts` struct (Clone, Debug):
  - `target_branch: Option<String>`
  - `strategy: Option<MergeOrderStrategy>`
  - `verbose: bool`
  - `no_ai: bool`
  - `json: bool`

- `OrchestrationReport` struct (Serialize, Clone, Debug):
  - `run_id: String`
  - `manifest_name: String`
  - `session_results: HashMap<String, SessionRunState>`
  - `merge_report: Option<MergeReport>` (from existing merge types)
  - `elapsed_secs: f64`
  - `outcome: RunPhase` (Complete or Failed)

**3. Create `crates/smelt-core/src/orchestrate/dag.rs`** with:

- `type SessionDag = petgraph::graph::DiGraph<String, ()>` — nodes are session names

- `build_dag(manifest: &Manifest) -> crate::Result<SessionDag>`:
  - Add nodes for each session (node weight = session name)
  - Add edges from explicit `depends_on` (edge from dependency to dependent)
  - If `parallel_by_default == false`, add implicit sequential chain edges for sessions without explicit `depends_on`
  - Validate with `petgraph::algo::is_cyclic_directed` — return `SmeltError::DependencyCycle` if cyclic
  - Return the validated DAG

- `ready_set(dag: &SessionDag, completed: &HashSet<NodeIndex>, in_flight: &HashSet<NodeIndex>, skipped: &HashSet<NodeIndex>) -> Vec<NodeIndex>`:
  - Return nodes where:
    - Not in completed, in_flight, or skipped
    - All incoming neighbors are in completed OR skipped
  - This finds sessions whose dependencies are satisfied

- `mark_skipped_dependents(dag: &SessionDag, failed_node: NodeIndex, skipped: &mut HashSet<NodeIndex>)`:
  - BFS/DFS from failed_node through outgoing edges
  - Mark all reachable nodes (transitive dependents) as skipped
  - Used when failure_policy is SkipDependents

- `node_by_name(dag: &SessionDag, name: &str) -> Option<NodeIndex>`:
  - Find the node index for a session name

- Unit tests:
  - `build_dag_parallel_sessions` — 3 independent sessions, no edges
  - `build_dag_linear_chain` — A->B->C with depends_on
  - `build_dag_diamond` — A->{B,C}->D pattern
  - `build_dag_implicit_sequential` — parallel_by_default=false creates chain
  - `build_dag_mixed_explicit_implicit` — some sessions with depends_on, some without, parallel_by_default=false
  - `ready_set_returns_roots` — initial ready set is all root nodes
  - `ready_set_after_completion` — completing a root unlocks dependents
  - `mark_skipped_dependents_transitive` — skipping A skips B and C when A->B->C
  - `mark_skipped_dependents_partial` — only direct dependents of failed node, not siblings
