<objective>
Implement the merge ordering algorithms (completion-time and file-overlap) and integrate them into MergeRunner. The greedy file-overlap algorithm picks sessions with least overlap against the already-merged file set. Strategy resolution follows CLI > manifest > default precedence.

Purpose: This is the core intelligence — the ordering algorithms that minimize merge conflicts. Without this, all sessions merge in manifest order regardless of file overlap patterns.
Output: ordering.rs module with algorithms, MergeRunner integration, MergePlan type for Plan 03's display.
</objective>

    2. Create `crates/smelt-core/src/merge/ordering.rs`:
       ```rust
       //! Merge ordering algorithms.

       use std::collections::HashSet;

       use crate::merge::types::{
           MergeOrderStrategy, MergePlan, PairwiseOverlap, SessionPlanEntry,
       };
       ```

       Implement the following functions:

       a. `CompletedSession` — the existing private struct in mod.rs needs a `changed_files` field. Add `pub(crate) changed_files: HashSet<String>` and `pub(crate) original_index: usize` to it. Make the struct `pub(crate)` so ordering.rs can access it.

       b. `pub(crate) fn order_sessions(sessions: Vec<CompletedSession>, strategy: MergeOrderStrategy) -> (Vec<CompletedSession>, MergePlan)`:
          - Dispatches to the appropriate ordering function.
          - Returns both the reordered sessions and a MergePlan for display.

       c. `fn completion_time_order(sessions: Vec<CompletedSession>) -> (Vec<CompletedSession>, MergePlan)`:
          - Preserves the input order (sessions are already in manifest order).
          - Builds a MergePlan with `fell_back: false`, empty pairwise_overlaps.

       d. `fn file_overlap_order(sessions: Vec<CompletedSession>) -> (Vec<CompletedSession>, MergePlan)`:
          - Compute pairwise overlaps: for every pair (i, j) where i < j, compute `session_i.changed_files.intersection(&session_j.changed_files)`.
          - Check if overlap strategy can meaningfully differentiate: if ALL pairwise overlaps are 0 OR ALL are equal, set `fell_back: true` and return manifest order with a note.
          - Greedy algorithm:
            ```
            merged_files: HashSet<String> = empty
            remaining: Vec<CompletedSession> = all sessions
            ordered: Vec<CompletedSession> = empty

            while !remaining.is_empty():
                pick session with MINIMUM overlap against merged_files
                    overlap = session.changed_files.intersection(&merged_files).count()
                    tiebreak: lower original_index wins
                move picked session from remaining to ordered
                merged_files.extend(picked.changed_files)
            ```
          - Build MergePlan with pairwise_overlaps populated and the ordered session list.

       e. `pub(crate) fn compute_pairwise_overlaps(sessions: &[CompletedSession]) -> Vec<PairwiseOverlap>`:
          - For each pair (i, j) where i < j, compute overlap.
          - Return all pairs (including zero-overlap pairs for completeness).
