<objective>
Refactor the merge loop to support conflict handling via the ConflictHandler trait. When a squash merge produces conflicts, scan for markers, invoke the handler, and execute the chosen action (resolve/skip/abort). Add resume detection and a NoopConflictHandler for backward compatibility.

Purpose: This is the core behavioral change — transforming merge conflicts from fatal errors into recoverable situations. The handler trait allows Plan 03 to plug in the interactive CLI handler without modifying core logic.
Output: ConflictHandler trait, modified merge_sessions() with conflict resolution loop, NoopConflictHandler, resume detection, MergeAborted error variant.
</objective>

       - Define the `ConflictHandler` trait (use `std::future::Future` since it's already imported for GitOps):
         ```rust
         /// Trait for handling merge conflicts during the merge loop.
         ///
         /// Core provides [`NoopConflictHandler`] which propagates conflicts as errors.
         /// The CLI crate provides an interactive handler using dialoguer.
         pub trait ConflictHandler: Send + Sync {
             /// Called when a merge conflict is detected.
             ///
             /// `session_name`: the session that produced the conflict.
             /// `files`: list of conflicting file paths.
             /// `scan`: conflict marker scan results (hunk positions, line counts).
             ///
             /// Returns the user's chosen action.
             fn handle_conflict(
                 &self,
                 session_name: &str,
                 files: &[String],
                 scan: &conflict::ConflictScan,
             ) -> impl Future<Output = Result<ConflictAction>> + Send;
         }
         ```

       - Implement `NoopConflictHandler`:
         ```rust
         /// A conflict handler that always propagates the conflict as an error.
         /// Used in tests and non-interactive contexts.
         pub struct NoopConflictHandler;

         impl ConflictHandler for NoopConflictHandler {
             async fn handle_conflict(
                 &self,
                 session_name: &str,
                 files: &[String],
                 _scan: &conflict::ConflictScan,
             ) -> Result<ConflictAction> {
                 Err(SmeltError::MergeConflict {
                     session: session_name.to_string(),
                     files: files.to_vec(),
                 })
             }
         }
         ```

    3. Refactor `MergeRunner::run()`:
       - Change signature to accept a generic handler:
         ```rust
         pub async fn run<H: ConflictHandler>(&self, manifest: &Manifest, opts: MergeOpts, handler: &H) -> Result<MergeReport>
         ```
       - Pass `handler` through to `merge_sessions()`.
       - In the success path, compute `sessions_conflict_skipped` and `sessions_resolved` from the session results' `resolution` field.
       - Handle `SmeltError::MergeAborted` in the error match the same way as other errors (rollback).

    4. Refactor `merge_sessions()`:
       - Change signature to accept handler:
         ```rust
         async fn merge_sessions<H: ConflictHandler>(
             &self,
             sessions: &[CompletedSession],
             temp_path: &Path,
             target_branch: &str,
             handler: &H,
             verbose: bool,
         ) -> Result<Vec<MergeSessionResult>>
         ```
       - After `merge_squash` fails with `SmeltError::MergeConflict`:
         a. Get unmerged files via `self.git.unmerged_files(temp_path)`.
         b. Call `conflict::scan_files_for_markers(temp_path, &files)`.
         c. Call `handler.handle_conflict(session_name, &files, &scan)`.
         d. Match on the returned `ConflictAction`:
            - **Resolved**: Enter a validation loop:
              - Re-scan for markers via `scan_files_for_markers`.
              - If markers remain, call `handle_conflict` again (re-prompt).
              - If no markers, auto-stage ALL files (`git add .` equivalent — use `self.git.add(temp_path, &["."])`) and commit with `[resolved: manual]` suffix.
              - Set `resolution: Some(ResolutionMethod::Manual)`.
            - **Skip**: `self.git.reset_hard(temp_path, "HEAD")`. Push a result with `resolution: Some(ResolutionMethod::Skipped)`, zero diff stats, empty commit hash.
            - **Abort**: Return `Err(SmeltError::MergeAborted { session })`.
         e. Use `.map_err()` on the initial `merge_squash` call to catch only `MergeConflict`, letting other errors propagate normally.

    5. **Important constraint on `add` for resolved conflicts:**
       The current `GitOps::add` requires explicit file paths and panics on empty. For conflict resolution, all modified files should be staged. Either:
       - Call `self.git.add(temp_path, &["."])` to stage everything in the worktree (this is safe because it's a temp worktree dedicated to this merge).
       - Or collect the files from unmerged_files and add them explicitly.
       Choose the `"."` approach since it's a dedicated temp worktree. This may require relaxing the `add` assertion to allow `"."` — check the implementation. If the assertion is `assert!(!paths.is_empty())`, `&["."]` will work fine.

    6. Update ALL existing tests in `merge/mod.rs` to pass `&NoopConflictHandler` to `runner.run()`. This preserves their behavior exactly — NoopConflictHandler propagates MergeConflict as before.

       - Assert: `report.sessions_conflict_skipped` contains the skipped session name.
       - Assert: target branch exists (not rolled back).

    8. Add a new test `test_merge_conflict_abort`:
       - Same conflicting setup.
       - Create an `AbortConflictHandler` (test-only) that returns `ConflictAction::Abort`.
       - Assert: merge returns `SmeltError::MergeAborted`.
       - Assert: target branch does NOT exist (rolled back).
