<objective>
Build the interactive CLI conflict handler and wire it into the merge command. When a conflict occurs, the user sees a summary of conflicting files with line ranges, edits files externally, then signals resolve/skip/abort via dialoguer. Add --verbose flag support and update CLI output to include resolution status.

Purpose: This completes the human fallback resolution feature. Users can now recover from merge conflicts interactively instead of having the entire merge abort. This is the safety net required before Phase 7 (AI conflict resolution).
Output: Interactive conflict handler, --verbose flag, updated merge CLI output with resolution status, integration tests.
</objective>

       Implementation detail for ResolveConflictHandler:
       ```rust
       struct ResolveConflictHandler {
           work_dir: PathBuf,
       }

       impl ConflictHandler for ResolveConflictHandler {
           async fn handle_conflict(
               &self,
               _session_name: &str,
               files: &[String],
               _scan: &conflict::ConflictScan,
               work_dir: &Path,
           ) -> Result<ConflictAction> {
               // Resolve by writing a clean version of each conflicting file
               for file in files {
                   let path = work_dir.join(file);
                   let content = std::fs::read_to_string(&path).unwrap_or_default();
                   // Simple resolution: keep everything that's not a conflict marker
                   let resolved: String = content
                       .lines()
                       .filter(|l| {
                           !l.starts_with("<<<<<<<")
                               && !l.starts_with("=======")
                               && !l.starts_with(">>>>>>>")
                       })
                       .collect::<Vec<_>>()
                       .join("\n");
                   std::fs::write(&path, format!("{resolved}\n")).unwrap();
               }
               Ok(ConflictAction::Resolved)
           }
       }
       ```
