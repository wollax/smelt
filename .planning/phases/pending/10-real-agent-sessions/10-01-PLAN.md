  <action>
Create `crates/smelt-core/src/session/agent.rs` with:

1. **AgentExecutor struct:**
   ```rust
   pub struct AgentExecutor {
       worktree_path: PathBuf,
       log_path: PathBuf,
       timeout: Option<Duration>,
   }
   ```

2. **Constructor:** `new(worktree_path, log_path, timeout)` — no git dependency needed since we just spawn a process.

3. **Prompt construction:** Private method `build_prompt(session_name, task, file_scope) -> String` that assembles a task prompt from the session's `task` description and optional `file_scope` hints. Keep it simple for v0.1.0 — task description + file scope list + instruction to commit work.

4. **CLAUDE.md injection:** Private method `inject_claude_md(session_name, task, file_scope) -> Result<()>` that:
   - Checks if worktree already has a `CLAUDE.md` at root
   - If yes: write to `.claude/CLAUDE.md` instead (creating `.claude/` dir)
   - If no: write to `CLAUDE.md` at worktree root
   - Content: session name, task description, file scope constraints, instruction to commit work with meaningful messages
   - Include guidance: "Stay within the assigned file scope. Commit your work with descriptive messages."

5. **settings.json injection:** Private method `inject_settings(model: Option<&str>) -> Result<()>` that:
   - Creates `.claude/` directory in worktree if it doesn't exist
   - Writes `.claude/settings.json` with permissions configuration for headless execution
   - Use `--dangerously-skip-permissions` as the primary mechanism; settings.json is defense-in-depth
   - Pin model if provided via `"model"` field in settings

6. **execute method:**
   ```rust
   pub async fn execute(
       &self,
       session_name: &str,
       task: &str,
       file_scope: Option<&[String]>,
       model: Option<&str>,
       cancel: CancellationToken,
   ) -> Result<SessionResult>
   ```
   Implementation:
   - Call `inject_claude_md()` and `inject_settings()`
   - Build prompt via `build_prompt()`
   - Spawn `claude` via `tokio::process::Command`:
     - Args: `-p`, `<prompt>`, `--dangerously-skip-permissions`, `--output-format`, `json`
     - If model provided: `--model`, `<model>`
     - `current_dir(&self.worktree_path)`
     - `.stdout(Stdio::piped())`, `.stderr(Stdio::piped())`
     - `.process_group(0)` and `.kill_on_drop(true)`
   - Extract PID via `child.id()`
   - Use `tokio::select!` with three arms:
     - `cancel.cancelled()` -> kill_process_group(pid), return Killed
     - `timeout(deadline, child.wait_with_output())` on timeout -> kill_process_group(pid), return TimedOut
     - Normal completion -> map exit code to outcome
   - If no timeout configured, use just cancel + wait (no timeout arm)
   - Write stdout+stderr to `self.log_path`
   - Check for commits: use rev_list_count or a simpler heuristic (check exit code — Claude Code exits 0 on success)
   - Map to SessionResult:
     - Exit code 0 = Completed (has_commits determined by checking `git rev-list --count HEAD` after vs before — but for simplicity, assume exit 0 means work was done, set has_commits=true)
     - Exit code non-zero = Failed with stderr as failure_reason
     - Timeout = TimedOut
     - Cancelled = Killed

7. **kill_process_group helper:** Private async fn that sends SIGTERM to -pgid (reuse the pattern from ProcessGroup but for async context). After SIGTERM, wait briefly for process exit.

8. **resolve_claude_binary:** Public function `resolve_claude_binary() -> Result<PathBuf>` that:
   - Uses `which::which("claude")` to find the binary
   - Returns SmeltError if not found (add a new variant or reuse SessionError)

Add to `SmeltError` in error.rs:
- `AgentNotFound` variant: `#[error("'claude' CLI not found on $PATH. Install Claude Code to use real agent sessions.")]`

Add to `session/mod.rs`:
- `pub mod agent;`
- `pub use agent::AgentExecutor;`

**Important implementation notes:**
- Do NOT use std::process — use tokio::process throughout for async compatibility
- The `--dangerously-skip-permissions` flag is essential for headless execution
- `--output-format json` gives structured output (parse with serde_json::Value for v0.1.0)
- Do NOT add new workspace dependencies — everything needed is already available (tokio, libc, serde_json, which)
- steps_completed field in SessionResult: set to 0 for agent sessions (not meaningful for non-scripted execution)
