Modify `Orchestrator::execute_sessions()` in `crates/smelt-core/src/orchestrate/executor.rs`:

1. **Add preflight check:** Add a method `check_agent_preflight(manifest: &Manifest) -> Result<Option<PathBuf>>` on Orchestrator that:
   - Scans manifest sessions for any that have `script: None` (i.e., agent sessions)
   - If any exist, calls `resolve_claude_binary()` and returns the PathBuf
   - If none exist, returns None (skip the check — scripted-only manifests don't need claude)
   - Call this in `run()` before Phase 1 (worktree creation). On error, return immediately.

2. **Modify the session dispatch block** in `execute_sessions()`. Currently the code has:
   ```rust
   } else {
       // No script — immediately complete
       Ok(SessionRunState::Completed {
           duration_secs: timer.elapsed().as_secs_f64(),
       })
   };
   ```

   Replace this `else` branch with agent execution:
   - Create an `AgentExecutor::new(claude_binary, worktree_path, log_path, timeout)` (using the binary path from preflight)
     - `timeout`: derive from `session_def.timeout_secs` -> `Duration::from_secs(t)`
     - `log_path`: already available as `log_path` in the spawn closure
   - Resolve the task: `session_def.task` or read from `session_def.task_file` (task_file: read the file content as the task string)
   - Call `executor.execute(session_name, task, file_scope, model, child_cancel)`
     - `file_scope`: `session_def.file_scope.as_deref()` — convert `Option<Vec<String>>` to `Option<&[String]>`
     - `model`: None for v0.1.0 (or read from env config if trivially available — Claude's discretion)
   - Map `SessionResult` to `SessionRunState` the same way scripted sessions are mapped:
     - Completed -> SessionRunState::Completed
     - Failed -> SessionRunState::Failed
     - TimedOut -> SessionRunState::Failed with reason "session timed out"
     - Killed -> SessionRunState::Cancelled

3. **Pass claude_binary_path** into the spawn closure. The preflight returns `Option<PathBuf>`. Clone it into each agent spawn. AgentExecutor already takes `claude_binary: PathBuf` in its constructor (defined in Plan 10-01), so pass the resolved path directly.

4. **Ensure task_file resolution happens before the async spawn boundary.** Before `join_set.spawn()`, resolve the task string:
   ```rust
   let task_content = if let Some(tf) = &session_def.task_file {
       tokio::fs::read_to_string(tf).await?
   } else {
       session_def.task.clone()
   };
   ```
   Then move `task_content` into the async closure via the `async move { ... }` block. This avoids blocking reads inside the spawned task and ensures the file is resolved while we still have the await context.

5. **Re-export AgentExecutor from lib.rs** if not already accessible (check if session module re-exports are sufficient).

**Important:**
- Do NOT change the scripted session path — only modify the `else` (script=None) branch
- The `git` clone is already available in the spawn closure — pass it if AgentExecutor needs it
- Keep the same JoinSet<(String, Result<SessionRunState, String>)> signature
- Maintain the same error handling pattern (map errors to SessionRunState::Failed)

**SessionRunner (runner.rs):**

Modify `SessionRunner::run_manifest()` to dispatch to `AgentExecutor` when `session.script` is None:

1. Replace the current `None` branch (which returns immediate Completed with no commits) with:
   - Resolve `claude` binary via `resolve_claude_binary()` (or accept it as a parameter)
   - Create AgentExecutor with the worktree path and a default log path
   - Call `execute()` with task, file_scope, and a new CancellationToken (no orchestrator-level cancel in SessionRunner — create a dummy token)
   - Map the SessionResult through the same state update logic

   Note: SessionRunner is used by `smelt session run` (standalone, non-orchestrated). For v0.1.0, it's acceptable to have a simpler integration here — no timeout (or use session_def.timeout_secs), no cancellation (or a never-cancelled token).
