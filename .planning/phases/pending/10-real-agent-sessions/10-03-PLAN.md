Create integration tests. Check the existing test structure first — if tests live in `crates/smelt-core/tests/` or `tests/`, follow the established pattern.

**Tests to write (all marked `#[ignore]` since they require `claude` on PATH):**

1. **test_agent_executor_spawns_claude_and_completes**:
   - Create a temp git repo with initial commit and .smelt/ initialized
   - Create a worktree
   - Create an AgentExecutor with the worktree path
   - Call execute() with a simple task: "Create a file called hello.txt with the content 'Hello from Claude'"
   - Assert: outcome is Completed, has_commits is true
   - Assert: the worktree has at least one new commit
   - Assert: log file was written

2. **test_agent_executor_timeout_kills_process**:
   - Create a temp git repo + worktree
   - Create AgentExecutor with timeout = 5 seconds
   - Call execute() with a task that would take longer (e.g., "Implement a complete web framework from scratch with 100+ files")
   - Assert: outcome is TimedOut
   - Assert: process is not running (no zombie)

3. **test_agent_executor_cancellation_kills_process**:
   - Create a temp git repo + worktree
   - Create AgentExecutor with no timeout
   - Spawn execute() in a tokio task
   - Wait 2 seconds, then cancel the token
   - Assert: outcome is Killed
   - Assert: process is not running

4. **test_orchestrator_two_agent_sessions_merge**:
   - Create a temp git repo
   - Build a manifest with 2 agent sessions (no script field):
     - Session "add-greeting": task = "Create src/greeting.rs with a public greet() function that returns 'Hello, World!'"
     - Session "add-farewell": task = "Create src/farewell.rs with a public farewell() function that returns 'Goodbye, World!'"
   - Run the full Orchestrator.run()
   - Assert: OrchestrationReport shows both sessions Completed
   - Assert: merge report shows successful merge
   - Assert: target branch contains both src/greeting.rs and src/farewell.rs

5. **test_agent_session_injects_claude_md**:
   - Create a temp git repo + worktree
   - Run AgentExecutor
   - After execution, verify CLAUDE.md (or .claude/CLAUDE.md) exists in the worktree
   - Verify it contains the session name and task description

6. **test_agent_session_injects_settings_json**:
   - Create a temp git repo + worktree
   - Run AgentExecutor
   - After execution, verify .claude/settings.json exists and is valid JSON

7. **test_mixed_scripted_and_agent_sessions** (if `claude` is on PATH):
   - Manifest with one scripted session and one agent session
   - Both complete and merge successfully
   - Proves backward compatibility

**Tests that do NOT require `claude` (run always):**

8. **test_agent_not_installed_returns_clear_error**:
   - Set PATH to empty or a directory without `claude`
   - Call `resolve_claude_binary()`
   - Assert: returns AgentNotFound error with helpful message

9. **test_orchestrator_preflight_skips_when_all_scripted**:
   - Create a scripted-only manifest
   - Verify preflight does not fail even if `claude` is not installed
