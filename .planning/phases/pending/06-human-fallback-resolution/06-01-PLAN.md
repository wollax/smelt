    1. Create `crates/smelt-core/src/merge/conflict.rs`:
       - Define `ConflictHunk`:
         ```rust
         /// A contiguous region of conflict markers in a file.
         #[derive(Debug, Clone)]
         pub struct ConflictHunk {
             /// 1-based line number of the opening <<<<<<< marker.
             pub start_line: usize,
             /// 1-based line number of the closing >>>>>>> marker.
             pub end_line: usize,
         }
         ```

       - Define `ConflictScan`:
         ```rust
         /// Result of scanning a file for conflict markers.
         #[derive(Debug, Clone)]
         pub struct ConflictScan {
             /// Whether any conflict markers were found.
             pub has_markers: bool,
             /// Detected conflict hunks (regions between <<<<<<< and >>>>>>>).
             pub hunks: Vec<ConflictHunk>,
             /// Total number of lines within conflict regions.
             pub total_conflict_lines: usize,
         }
         ```

       - Implement `scan_conflict_markers(content: &str) -> ConflictScan`:
         - Iterate lines with enumerate (1-based).
         - Track state: looking for `<<<<<<<`, then `=======`, then `>>>>>>>`.
         - A valid hunk requires all three markers in sequence.
         - Count total_conflict_lines as end_line - start_line + 1 for each hunk.
         - Set has_markers to true if any hunk is found.
         - Be permissive: if a `<<<<<<<` is found but no closing `>>>>>>>` before EOF or another `<<<<<<<`, discard the partial hunk. This avoids false positives.

       - Implement `scan_files_for_markers(work_dir: &Path, files: &[String]) -> ConflictScan`:
         - Reads each file, runs scan_conflict_markers on its content.
         - Aggregates results into a single ConflictScan (merge all hunks, sum lines, has_markers = any file has markers).
         - Files that cannot be read are silently skipped (they may be binary or deleted).

       - Add unit tests:
         - Test with no markers -> has_markers false, empty hunks.
         - Test with one complete hunk -> correct start/end lines.
         - Test with multiple hunks -> all detected.
         - Test with partial/malformed markers -> no hunks (graceful).
         - Test with nested markers -> only outermost detected.

    2. In `crates/smelt-core/src/merge/mod.rs`:
       - Add `pub mod conflict;` to the module declarations.
       - Add `pub use conflict::{ConflictHunk, ConflictScan, scan_conflict_markers};` (or keep internal as `pub(crate)` and only expose what's needed).

    3. In `crates/smelt-core/src/git/mod.rs`, add to the `GitOps` trait:

       /// List commit subjects in a range.
       /// Uses `git log --format="%s" <range>`.
       fn log_subjects(&self, range: &str) -> impl Future<Output = Result<Vec<String>>> + Send;
       ```

    4. In `crates/smelt-core/src/git/cli.rs`, implement for `GitCli`:

       async fn log_subjects(&self, range: &str) -> Result<Vec<String>> {
           let output = self.run(&["log", "--format=%s", range]).await?;
           if output.is_empty() {
               return Ok(Vec::new());
           }
           Ok(output.lines().map(|l| l.to_string()).collect())
       }
       ```
