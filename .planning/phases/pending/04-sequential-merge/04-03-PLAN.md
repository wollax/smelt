  <action>
    1. Create `crates/smelt-cli/src/commands/merge.rs`:

       Implement `pub async fn execute_merge(git: GitCli, repo_root: PathBuf, manifest_path: &str, target: Option<String>) -> anyhow::Result<i32>`:

       a. Load the manifest: `Manifest::load(Path::new(manifest_path))?`
       b. Create MergeRunner and MergeOpts
       c. Print start message to stderr: `eprintln!("Merging sessions from manifest '{}'...", manifest.manifest.name)`
       d. Call `runner.run(&manifest, opts)`
       e. On success (MergeReport):
          - Print per-session results to stderr during merge (this requires hooking into the merge loop... BUT MergeRunner doesn't have callbacks).
          - ALTERNATIVE: Print summary AFTER merge completes. The MergeReport has all the data.
          - To stderr (progress):
            ```
            Merged N session(s) into 'smelt/merge/<name>'
            Skipped M session(s): <names>
            ```
          - To stdout (summary): For each session in sessions_merged:
            ```
            <session-name>:
              <files_changed> file(s) changed, <insertions> insertion(s), <deletions> deletion(s)
            ```
          - Return Ok(0)
       f. On error:
          - Match specific error types:
            - `SmeltError::MergeConflict { session, files }`:
              ```
              eprintln!("Error: merge conflict in session '{session}'");
              eprintln!("Conflicting files:");
              for f in &files { eprintln!("  {f}"); }
              eprintln!("Target branch rolled back. Session worktrees preserved for inspection.");
              ```
              Return Ok(1)
            - `SmeltError::MergeTargetExists { branch }`:
              `eprintln!("Error: {err}")` — Return Ok(1)
            - `SmeltError::NoCompletedSessions`:
              `eprintln!("Error: {err}")` — Return Ok(1)
            - Other errors: propagate with `?`

    2. Update `crates/smelt-cli/src/commands/mod.rs`:
       Add `pub mod merge;`

    3. Update `crates/smelt-cli/src/main.rs`:
       - Add `Merge` variant to `Commands` enum:
         ```rust
         /// Merge session outputs into a single branch
         Merge {
             /// Path to the session manifest file
             manifest: String,
             /// Override target branch name
             #[arg(long)]
             target: Option<String>,
         },
         ```
       - Add match arm in `run()`:
         ```rust
         Some(Commands::Merge { manifest, target }) => {
             let git = GitCli::new(git_binary, repo_root.clone());
             commands::merge::execute_merge(git, repo_root, &manifest, target).await
         }
         ```

    NOTE on progress output: Ideally, we'd print `[1/3] Merging "session-name"... ok` in real-time as each session merges. Since MergeRunner doesn't expose a callback/channel mechanism, the simplest approach for Phase 4 is to print summary output after completion. If real-time progress is needed, either:
    - Add a progress callback to MergeRunner::run() (adds complexity)
    - Use tracing events that the CLI subscribes to (elegant but complex)
    - Print post-hoc from MergeReport (simplest, chosen for Phase 4)

    For Phase 4, print post-hoc progress from MergeReport to stderr:
    ```
    for (i, result) in report.sessions_merged.iter().enumerate() {
        eprintln!("[{}/{}] Merged '{}'", i + 1, report.sessions_merged.len(), result.session_name);
    }
    ```
