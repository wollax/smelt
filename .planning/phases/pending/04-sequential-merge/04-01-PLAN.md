    1. Add 8 new methods to the `GitOps` trait in `git/mod.rs`:

       ```rust
       /// Find the merge-base (common ancestor) of two refs.
       fn merge_base(&self, ref_a: &str, ref_b: &str) -> impl Future<Output = Result<String>> + Send;

       /// Create a new branch at `start_point` without checking it out.
       fn branch_create(&self, branch_name: &str, start_point: &str) -> impl Future<Output = Result<()>> + Send;

       /// Perform a squash merge of `source_ref` into the current branch of `work_dir`.
       /// Returns Ok(()) on clean merge. Returns SmeltError::MergeConflict on conflict (with file list).
       /// Returns SmeltError::GitExecution on other git errors.
       fn merge_squash(&self, work_dir: &Path, source_ref: &str, session_name: &str) -> impl Future<Output = Result<()>> + Send;

       /// Check out an existing branch into a new worktree path (no -b flag).
       fn worktree_add_existing(&self, path: &Path, branch_name: &str) -> impl Future<Output = Result<()>> + Send;

       /// List unmerged (conflicting) files in `work_dir`.
       /// Uses `git diff --name-only --diff-filter=U`.
       fn unmerged_files(&self, work_dir: &Path) -> impl Future<Output = Result<Vec<String>>> + Send;

       /// Hard reset HEAD in `work_dir` to `target_ref`.
       fn reset_hard(&self, work_dir: &Path, target_ref: &str) -> impl Future<Output = Result<()>> + Send;

       /// Resolve a ref (branch name, HEAD, etc.) to a full commit hash.
       fn rev_parse(&self, rev: &str) -> impl Future<Output = Result<String>> + Send;

       /// Get diff stats between two refs. Returns Vec of (insertions, deletions, filename).
       fn diff_numstat(&self, from_ref: &str, to_ref: &str) -> impl Future<Output = Result<Vec<(usize, usize, String)>>> + Send;
       ```

    2. Implement all 8 methods in `GitCli` (`git/cli.rs`):

       **`merge_base`**: Run `git merge-base <ref_a> <ref_b>` via `self.run()`. Return trimmed stdout (full SHA).

       **`branch_create`**: Run `git branch <branch_name> <start_point>` via `self.run()`. Return Ok(()).

       **`merge_squash`**: Use a raw `tokio::process::Command` (NOT `run_in`) to run `git -C <work_dir> merge --squash <source_ref>`. Inspect exit code and stderr directly:
       - Exit 0 → return Ok(())
       - Exit 1 AND stderr contains "CONFLICT" → call `self.unmerged_files(work_dir)` internally, then return `Err(SmeltError::MergeConflict { session: session_name.to_string(), files })`
       - Any other non-zero exit → return `Err(SmeltError::GitExecution { command, stderr, exit_code })`
       This gives MergeRunner domain-specific errors without needing to distinguish conflict from general failure externally. The `session_name` parameter is passed through from the caller.

       **`worktree_add_existing`**: Run `git worktree add <path> <branch_name>` via `self.run()` (no `-b` flag). Return Ok(()).

       **`unmerged_files`**: Run `git diff --name-only --diff-filter=U` via `self.run_in(work_dir, ...)`. Split stdout by newlines into Vec<String>. Return empty vec if stdout is empty.

       **`reset_hard`**: Run `git reset --hard <target_ref>` via `self.run_in(work_dir, ...)`. Return Ok(()).

       **`rev_parse`**: Run `git rev-parse <rev>` via `self.run()`. Return trimmed stdout (full SHA). This differs from `head_short` which uses `--short`.

       **`diff_numstat`**: Run `git diff --numstat <from_ref> <to_ref>` via `self.run()`. Parse each line as `<insertions>\t<deletions>\t<filename>`. Handle binary files where insertions/deletions are `-` (use 0 for those). Return `Vec<(usize, usize, String)>`.

    3. Unit tests in `cli.rs` (extend `setup_test_repo`):

       - **test_merge_squash_conflict**: Create two branches that edit the same file differently. Merge_squash one into the other. Verify it returns Err(SmeltError::MergeConflict) with the conflicting file in the file list. (No separate unmerged_files call needed — merge_squash returns the conflict info directly.)
       - **test_reset_hard**: Make changes in a worktree, reset_hard to HEAD. Verify worktree is clean.
       - **test_rev_parse**: Verify rev_parse("HEAD") returns a 40-char hex hash.
       - **test_diff_numstat**: Create a branch with a commit adding a file, verify diff_numstat shows correct insertions/deletions.
       - **test_unmerged_files_empty_when_clean**: Call unmerged_files on a clean worktree, verify empty vec.
       - **test_worktree_add_existing**: Create a branch, then use worktree_add_existing to check it out into a new path. Verify the worktree exists and is on the correct branch.

       IMPORTANT for merge_squash test: After `git merge --squash`, the changes are staged but NOT committed. You need to commit them manually. The test should verify this flow:
       1. Create worktree on target branch
       2. merge_squash(worktree_path, source_branch)
       3. commit(worktree_path, "squash commit")
       4. Verify the commit exists
