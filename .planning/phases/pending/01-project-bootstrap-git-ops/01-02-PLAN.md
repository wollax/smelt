Implement the smelt-core library: error types, `GitOps` trait, `GitCli` shell-out implementation, `preflight()` sanity checks, and `init_project()` for `.smelt/` creation. This is the foundation all CLI commands and future phases build on.

<action>
1. Create `crates/smelt-core/src/error.rs` per Example 3 in research:
   - `SmeltError` enum with `#[non_exhaustive]`, deriving `Debug` and `thiserror::Error`
   - Variants: `GitNotFound`, `NotAGitRepo`, `GitExecution { operation, message }`, `AlreadyInitialized { path }`, `Io { operation, path, source }`
   - Add convenience constructor `SmeltError::io(operation, path, source)` for Io variant
   - `pub type Result<T> = std::result::Result<T, SmeltError>;`

2. Create `crates/smelt-core/src/git/mod.rs`:
   - Define `pub trait GitOps` with async methods:
     - `async fn repo_root(&self) -> Result<PathBuf>`
     - `async fn is_inside_work_tree(&self, path: &Path) -> Result<bool>`
     - `async fn current_branch(&self) -> Result<String>`
     - `async fn head_short(&self) -> Result<String>`
   - Define `pub fn preflight() -> Result<(PathBuf, PathBuf)>` (synchronous):
     - Use `which::which("git")` to find git binary, return `SmeltError::GitNotFound` if missing
     - Run `std::process::Command::new(&git_binary).args(["rev-parse", "--show-toplevel"])`
     - Return `SmeltError::NotAGitRepo` if exit code non-zero
     - Parse stdout as repo root PathBuf
     - Return `(git_binary, repo_root)` tuple
   - Re-export `GitCli` from cli module

3. Create `crates/smelt-core/src/git/cli.rs` per Example 6 in research:
   - `pub struct GitCli` with `git_binary: PathBuf` and `repo_root: PathBuf`
   - `GitCli::new(git_binary: PathBuf, repo_root: PathBuf) -> Self` constructor
   - Private `async fn run(&self, args: &[&str]) -> Result<String>` helper:
     - Uses `tokio::process::Command`, sets `current_dir` to repo_root
     - Returns trimmed stdout on success, `SmeltError::GitExecution` on failure
   - Implement `GitOps` for `GitCli`:
     - `repo_root()` â€” return cached `self.repo_root.clone()`
     - `is_inside_work_tree(path)` â€” run `rev-parse --is-inside-work-tree` with current_dir set to path
     - `current_branch()` â€” run `rev-parse --abbrev-ref HEAD`
     - `head_short()` â€” run `rev-parse --short HEAD`

4. Update `crates/smelt-core/src/lib.rs`:
   - Declare modules: `pub mod error;`, `pub mod git;`, `pub mod init;` (init module added in task 2)
   - Re-export key types: `pub use error::{SmeltError, Result};`, `pub use git::{GitOps, GitCli, preflight};`
</action>
