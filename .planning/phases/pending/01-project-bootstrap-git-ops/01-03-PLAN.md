<title>Implement CLI entry point and init command</title>
<files>

<action>
1. Create `crates/smelt-cli/src/commands/mod.rs`:
   - Define `pub mod init;`

2. Create `crates/smelt-cli/src/commands/init.rs`:
   - `pub fn execute(repo_root: &Path) -> anyhow::Result<i32>`:
     - Call `smelt_core::init_project(repo_root)`
     - On success: print "Initialized Smelt project in {path}" and hint "Run `git add .smelt/` to track Smelt configuration." Return `Ok(0)`
     - On `SmeltError::AlreadyInitialized`: print the error message to stderr. Return `Ok(1)`
     - Propagate other errors via `?`

3. Rewrite `crates/smelt-cli/src/main.rs` following Assay's pattern (Pattern 2, Pitfall 5):
   - Declare `mod commands;`
   - Define clap derive struct:
     ```rust
     #[derive(Parser)]
     #[command(name = "smelt", about = "Multi-agent orchestration for git", propagate_version = true, version)]
     struct Cli {
         /// Disable colored output
         #[arg(long)]
         no_color: bool,

         #[command(subcommand)]
         command: Option<Commands>,
     }

     #[derive(Subcommand)]
     enum Commands {
         /// Initialize a new Smelt project in the current repository
         Init,
     }
     ```
   - `async fn run() -> anyhow::Result<i32>`:
     - Parse CLI: `Cli::try_parse().unwrap_or_else(|e| e.exit())`
     - Handle `--no-color`: call `console::set_colors_enabled(false)` if flag set
     - Initialize tracing subscriber (basic, with env-filter from `SMELT_LOG` or `RUST_LOG`)
     - Call `smelt_core::preflight()` — on error, print to stderr and return exit code 1
     - Match on `cli.command`:
       - `Some(Commands::Init)` → call `commands::init::execute(&repo_root)`
       - `None` → check if `.smelt/` exists at `repo_root.join(".smelt")`:
         - Exists: print basic status (project name from dir, current branch via GitCli). Return `Ok(0)`
         - Not exists: print "Not a Smelt project. Run `smelt init` to get started." to stderr, then print help via `Cli::command().print_help()`. Return `Ok(1)`
   - `#[tokio::main] async fn main()`:
     - Match on `run().await`
     - Ok(code) → `std::process::exit(code)`
     - Err(e) → `eprintln!("Error: {e:#}")` then `std::process::exit(1)`
</action>

<action>
Create `crates/smelt-cli/tests/cli_integration.rs` using `assert_cmd` and `tempfile`:

1. Helper function `setup_git_repo() -> tempfile::TempDir`:
   - Create tempdir
   - Run `git init` in it
   - Run `git commit --allow-empty -m "initial"` to create HEAD
   - Return the TempDir (RAII cleanup)

2. Tests (each creates its own temp git repo):

   - `test_version` — run `smelt --version`, assert stdout contains "smelt" and exits 0

   - `test_help` — run `smelt --help`, assert stdout contains "init" and exits 0

   - `test_init_creates_smelt_dir` — run `smelt init` in temp repo, assert:
     - Exit code 0
     - stdout contains "Initialized"
     - `.smelt/config.toml` exists in temp repo
     - File contains "version = 1"

   - `test_init_already_initialized` — run `smelt init` twice, assert:
     - Second run exits non-zero
     - stderr contains "already" (case insensitive)

   - `test_no_args_outside_project` — run `smelt` in temp repo (no .smelt/), assert:
     - Exit code 1
     - stderr contains "Not a Smelt project"

   - `test_no_args_inside_project` — run `smelt init` then `smelt` with no args, assert:
     - Exit code 0
     - Stdout contains some status output

   - `test_no_color_flag` — run `smelt --no-color --help`, assert exits 0 (flag accepted)

   - `test_outside_git_repo` — run `smelt` in a tempdir that is NOT a git repo, assert:
     - Exit code 1
     - stderr contains "git" (error about not being in a git repo)

NOTE: For all tests, set `current_dir` on the `Command` to the temp repo. The binary is found via `assert_cmd::Command::cargo_bin("smelt")`. Set `env("HOME", tempdir)` to avoid git config issues. Also set `env("GIT_CONFIG_NOSYSTEM", "1")` and `env("GIT_AUTHOR_NAME", "Test")` etc. to isolate git from user config.
