# Milestone v0.1.0: Orchestration PoC

**Status:** SHIPPED 2026-03-11
**Goal:** Prove that Smelt can coordinate multiple agent sessions in worktrees and merge their outputs into a single coherent branch with AI-assisted conflict resolution.
**Phases:** 1-10
**Total Plans:** 30

## Overview

Smelt is a multi-agent orchestration layer that coordinates AI coding sessions in git worktrees and merges their outputs into coherent, verified branches. The v0.1.0 milestone proves the core loop: create worktrees, run agents (real and scripted), merge with AI-assisted conflict resolution, and report results — all using git as the coordination substrate.

## Milestone Summary

**Key Decisions:**

- Language: Rust — ecosystem alignment with Assay, single-binary distribution, tokio async runtime
- Git operations: Shell-out to git CLI behind SmeltGitOps trait; gix for reads where mature
- Build order: Human fallback before AI resolution (safety net first, optimization second)
- Sequential merge strategy (not octopus) — isolates conflicts to specific branch pairs
- Edition 2024 with rust-version 1.85 minimum
- AiProvider/ConflictHandler/GitOps traits use RPITIT (no async-trait crate)
- MergeConflictHandler enum dispatcher avoids RPITIT-no-dyn limitation
- RunState persists as JSON (not TOML) — matches serde tagged enum serialization
- Git-native state in .smelt/ directory (no external database)

**Issues Resolved:**

- WorktreeState status not updated after session execution (sessions stuck in Created)
- CLAUDECODE env var causing nested session detection errors in agent sessions
- PID 1 test assumption fixed for cross-platform compatibility
- TTY detection guard on stderr to prevent non-TTY IO errors
- Boxed AiInteractive variant to satisfy clippy large_enum_variant lint

**Technical Debt Incurred:**

- --yes flag on dirty worktree removal falls through to error instead of auto-confirming
- has_commits and steps_completed not populated in orchestrator agent path
- lib.rs does not re-export all public types (CLI uses sub-path imports)
- Cancellation integration test not implemented (unit-tested; shares code path with timeout)
